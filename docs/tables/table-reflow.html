Reflow table mode

The reflow table mode works by collapsing the table columns into a stacked presentation that looks like blocks of label/data pairs for each row. It's not possible to reliably style a table to look like this, so the plugin adds a bit of extra markup to make the display work.

Using reflow mode

The reflow responsive table mode is the simplest in terms of markup requirements because it only requires a table with a data-role="table" on the table element. There is no need to set the data-mode attribute since reflow is the default. For good measure, assign a unique ID to the table.

<table data-role="table" ID="my-table">

The plugin works by parsing the values (or abbr title) of the first row of header (TH) elements found in the table. For example, in the table above, the third table header is parsed to grab the contents ("Year"):

<th>Year</th>

The script then appends an element with the table header text before the contents of every cell. This is styled to look like a left aligned label for the cell when stacked at narrow widths. For example, for every table cell in the year column:

<td>1941</td>

An element is added before the text with a class of ui-table-cell-label:

<td><b class="ui-table-cell-label">Year</b>1941</td>

Setting media queries for the stack point

With our mobile-first approach, the base styles for reflow display the table in the stacked label/date format. This is done by hiding the table header rows, making each cell display:block and showing the label element injected into each cell. The label elements in each cell are styled like this:


.ui-table-reflow td .ui-table-cell-label,
.ui-table-reflow th .ui-table-cell-label { 
	display: block;
	min-width: 30%; 
	display: inline-block; 
}

At wider widths, a single media query switches to a standard table presentation by showing the table headers and hiding the labels in each cell.This media query should be copied and added to your custom stylesheet so you can modify the width breakpoint for the needs of data in your table. 

@media ( min-width: 40em ) {
	/* Show the table header rows */ 
	.ui-table-reflow td,
	.ui-table-reflow th,
	.ui-table-reflow tbody th,
	.ui-table-reflow tbody td,
	.ui-table-reflow thead td,
	.ui-table-reflow thead th {
		display: table-cell;
		margin: 0;
	}
	/* Hide the labels in each cell */ 
	.ui-table-reflow td .ui-table-cell-label,
	.ui-table-reflow th .ui-table-cell-label { 
		display: none;
	}
}

Choosing a breakpoint

In reflow mode, you want to pick at minimum width at which the entire table will fit comfortably within the screen as the breakpoint. On some sites, this may be 30em (480px) and on others, it could be as wide as 100em (1,600px) so there is no way for us to decide on a "standard breakpoint" that will work for everyone -- that's why this media query should be custom to your application.

All media query widths are set in ems so they respond to font size changes. To calculate a pixel withs in em's, divide the target width by 16 (pixels) then edit the 40em value in the example breakpoint.

It's possible that this breakpoint will vary for each table within a page if the content length and number of columns is very different across tables. If that's the case, create a set of table classes that can be used to apply one of a few standard table breakpoints.
